# 青途智伴AI生活助手 - 简历亮点

## 项目概述

**青途智伴AI生活助手**是一款面向大学生的全场景AI生活助手，整合课程管理、健康饮食、智能记账、RAG知识问答、AI笔记生成等功能。

- **技术栈**: Spring Boot 3.2.5 + Java 17 + MySQL + Elasticsearch + Redis + Quartz
- **前端**: UniApp (Vue3) - 微信小程序/H5/APP多端
- **AI能力**: 阿里云百炼Qwen Plus + RAG向量检索
- **核心亮点**: 多Agent编排、MCP多组件协同、RAG知识库、Skill可插拔体系

### 亮点一：多Agent智能编排系统

#### 描述
设计并实现了**多Agent智能编排系统**，通过意图分析器识别用户需求，并行调度专家Agent完成复杂任务，实现"输入即所得"的智能交互体验。

#### 架构图
```
用户消息: "今天天气怎么样？花了50吃饭，想记账"
                    │
                    ▼
┌─────────────────────────────────────────────────────────────┐
│              IntentAnalyzer (意图分析器)                      │
│  关键词匹配 + 语义理解 → 识别出2个意图                       │
│  1. weather → WeatherSpecialistAgent                       │
│  2. expense → ExpenseSpecialistAgent                       │
└─────────────────────────────────────────────────────────────┘
                    │
          ┌─────────┴─────────┐
          ▼                   ▼
┌─────────────────┐   ┌─────────────────┐
│ WeatherAgent    │   │ ExpenseAgent    │
│ 并行执行        │   │ 并行执行        │
│ 获取天气+穿搭   │   │ 记录消费+分类   │
└─────────────────┘   └─────────────────┘
          │                   │
          └─────────┬─────────┘
                    ▼
┌─────────────────────────────────────────────────────────────┐
│            ResultAggregator (结果聚合器)                      │
│  多个Agent结果 → 格式统一 → 自然语言返回                    │
│  "今天北京晴，18-25度，适合轻薄外套。已记录午餐消费50元。"   │
└─────────────────────────────────────────────────────────────┘
```

#### 核心价值
- **意图识别精准**: 关键词+语义双重判断，支持模糊表达
- **并行执行**: 多Agent同时工作，响应速度快
- **结果融合**: 统一格式输出，用户体验流畅
- **可扩展**: 新增Agent只需实现接口，自动纳入编排

#### 技术实现
```java
// 意图分析器
public List<AgentTask> analyzeIntent(String message) {
    List<AgentTask> tasks = new ArrayList<>();

    if (containsAny(message, "天气", "温度", "下雨")) {
        tasks.add(new AgentTask("weather", message));
    }
    if (containsAny(message, "记账", "花了", "消费")) {
        tasks.add(new AgentTask("expense", message));
    }
    if (containsAny(message, "课程", "上课")) {
        tasks.add(new AgentTask("course", message));
    }
    // ...

    return tasks;
}

// 编排器执行
public String orchestrate(String message) {
    List<AgentTask> tasks = analyzeIntent(message);

    // 并行执行多个Agent
    List<CompletableFuture<String>> futures = tasks.stream()
        .map(task -> CompletableFuture.supplyAsync(() ->
            specialistAgents.get(task.getType()).execute(task.getMessage())
        ))
        .collect(Collectors.toList());

    // 等待所有结果
    List<String> results = futures.stream()
        .map(CompletableFuture::join)
        .collect(Collectors.toList());

    // 聚合结果
    return resultAggregator.merge(results);
}
```

### 亮点二：RAG知识库 + 向量语义检索

#### 描述
构建了**基于Elasticsearch的RAG知识库系统**，结合向量语义检索和关键词混合检索，有效减少AI幻觉，让AI回答更精准、更有依据。

#### 架构流程
```
用户问题: "图书馆开放时间?"
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│           生成查询向量 (768维 text-embedding-v1)              │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│           ES混合检索 + 学校过滤                               │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ KNN k=5 向量检索 (cosine余弦相似度)                    │ │
│  │ + filter: schoolWebsite = "https://yjs.snut.edu.cn"   │ │
│  └───────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│           格式化检索结果                                     │
│  【标题】西北农林科技大学图书馆...\n开放时间为8:00-22:00     │
```

### 亮点三：MCP多组件协同调度架构

#### 描述
设计了**MCP (Model Context Protocol) 多组件协同调度中心**，统一调度定时任务、AI Agent、RAG检索、Skill技能等模块，实现业务逻辑的高内聚低耦合。

#### 架构图
```
┌─────────────────────────────────────────────────────────────┐
│                      MCP Server                             │
│  ┌─────────────────────────────────────────────────────┐ │
│  │              Tool Registry (工具注册中心)              │ │
│  │  school_* / rag_* / parse_* / ocr_* / weather_*     │ │
│  └─────────────────────────────────────────────────────┘ │
│                          │                                │
│                          ▼                                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │           Permission Layer (权限控制)                  │ │
│  │       PUBLIC / USER / ADMIN 三级权限隔离              │ │
│  └─────────────────────────────────────────────────────┘ │
│                          │                                │
│                          ▼                                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │              Audit Log (审计日志)                     │ │
│  │   工具调用、参数、返回值、执行耗时、追踪ID            │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ 学校工具  │   │ RAG工具   │   │ 文档工具  │
    │ school_  │   │ rag_     │   │ parse_   │
    │ search   │   │ search   │   │ document │
    └──────────┘   └──────────┘   └──────────┘
```

#### 可用工具 (12+)
| 工具名 | 功能 | 权限 |
|--------|------|------|
| school_search | 学校搜索 | USER |
| school_info | 学校信息 | USER |
| school_knowledge_upsert | 学校知识入库 | ADMIN |
| rag_search | RAG检索 | USER |
| rag_batch_upsert | 批量RAG入库 | ADMIN |
| rag_clear | 清空知识库 | ADMIN |
| parse_document | 文档解析 | USER |
| ocr_image | 图片OCR | USER |
| extract_schedule | 课表提取 | USER |

#### 核心价值
- **统一入口**: 所有工具通过统一MCP接口调用
- **权限分层**: PUBLIC/USER/ADMIN三级权限隔离
- **可审计**: 每次调用都有完整日志记录
- **可扩展**: 新增工具只需注册，无需修改调用方

#### 技术实现
```java
// 统一执行入口
public CommonResult<?> executeTool(Long userId, String toolName, Map<String, Object> params) {
    ToolHandlerData tool = toolRegistry.getTool(toolName);
    if (tool == null) {
        return CommonResult.fail("工具不存在: " + toolName);
    }

    // 权限检查
    if (!hasPermission(userId, tool.getPermission())) {
        return CommonResult.fail("权限不足");
    }

    // 执行 + 耗时统计
    long start = System.currentTimeMillis();
    Object result = tool.execute(params);
    long latency = System.currentTimeMillis() - start;

    // 审计日志
    auditLogService.log(userId, toolName, params, result, latency);

    return CommonResult.success(result);
}
```

### 亮点四：Skill可插拔技能体系

#### 描述
设计了**Skill可插拔体系**，将业务能力封装为独立技能模块，支持用户按需启用/禁用，实现功能的灵活组合和按需加载。

#### 八大技能模块
| 技能 | 功能 | 触发场景 |
|------|------|----------|
| 🌤️ 学习助手 | 课程笔记生成、考试重点 | 课后自动生成 |
| 📚 生活百科 | 校园信息查询 | 用户主动问答 |
| 💰 消费分析 | 月度账单解读 | 消费记录查看 |
| 🏃 健康管理 | BMI计算、饮食建议 | 健康档案更新 |
| 👔 天气穿搭 | 早安推送、穿衣建议 | 每日7:30定时 |
| 📅 课程管理 | 课表查询、提醒 | 课前15分钟 |
| 🔍 知识问答 | RAG检索、校园FAQ | 用户主动问答 |
| ✏️ 智能记账 | OCR识别、自动分类 | 账单上传 |

#### 设计模式
```java
// Skill接口定义
public interface Skill {
    String getName();
    String getDescription();
    boolean isEnabled(Long userId);
    Object execute(Object params);
}

// 用户技能配置表
@TableName("user_skill_config")
public class UserSkillConfig {
    private Long userId;
    private String skillKey;  // weather, course, cost...
    private Boolean enabled;
}

// 技能注册中心
@Component
public class SkillRegistry {
    private Map<String, Skill> skills = new HashMap<>();

    @PostConstruct
    public void init() {
        skills.put("weather", new WeatherSkill());
        skills.put("course", new CourseSkill());
        skills.put("expense", new ExpenseSkill());
        skills.put("calorie", new CalorieSkill());
        skills.put("note", new NoteSkill());
        skills.put("health", new HealthSkill());
        skills.put("chat", new ChatSkill());
        skills.put("search", new SearchSkill());
    }
}
```

#### 核心价值
- **用户自主**: 用户可选择开启/关闭哪些技能
- **按需加载**: 只加载用户启用的技能，减少资源消耗
- **独立演进**: 每个技能可独立开发测试，不影响其他模块
- **灵活组合**: 不同用户组合不同技能，满足个性化需求

## 技术深度总结

### 解决的问题
1. **多端适配**: UniApp一套代码，编译为微信小程序/H5/APP
2. **性能优化**: Caffeine(本地) + Redis(分布式) 二级缓存
3. **稳定性**: 分布式锁保证定时任务不重复执行
4. **可扩展**: MCP架构支持工具灵活扩展，Skill体系支持功能模块化

### 技术难点攻克
1. **向量检索**: ES KNN + 学校过滤 + 结果格式化 + 混合检索
2. **流式响应**: SSE实现AI生成内容的实时推送
3. **文档解析**: PDFBox + POI + AI智能解析多格式文档
4. **定时任务**: Quartz + Redis分布式锁实现精准提醒
5. **多Agent编排**: IntentAnalyzer + SpecialistAgent并行 + ResultAggregator聚合

### 架构思想
1. **MCP调度中心**: 统一工具管理，权限控制，审计日志
2. **多Agent编排**: 意图识别 + 并行执行 + 结果融合
3. **RAG知识库**: 私有知识 + 向量检索，减少AI幻觉
4. **Skill可插拔**: 业务能力模块化，用户按需启用
5. **二级缓存**: 本地缓存减少网络开销，Redis支持分布式
