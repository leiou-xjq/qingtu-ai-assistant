## 二、技术选型分析

### 2.1 后端框架选型

| 选项 | 选择 | 原因 |
|------|------|------|
| Spring Boot vs 裸机Java | **Spring Boot** | 生态完善、自动配置、开箱即用，大幅提升开发效率 |
| MyBatis-Plus vs JPA vs 裸机JDBC | **MyBatis-Plus** | Lambda查询语法简洁、内置分页插件、ActiveRecord模式 |

**为什么不选其他方案**:
- JPA: 复杂查询性能差，SQL优化困难，对复杂业务支持不够灵活
- 裸机JDBC: 大量模板代码，维护成本高，难以维护
- GraalVM: 性能好但生态不如Spring，排查问题困难

### 2.2 AI框架选型

| 选项 | 选择 | 原因 |
|------|------|------|
| LangChain4j + 自行封装 vs Spring AI vs 纯手写 | **LangChain4j + 自行封装** | LangChain4j提供Agent、Tool抽象框架，自行封装更轻量可控，避免框架过度封装带来的灵活性损失 |
| 通义千问 vs ChatGPT vs 文心一言 | **通义千问 (qwen-plus)** | 国产大模型、效果优秀(接近GPT-4)、成本低、国内合规无需翻墙 |

**为什么不选其他方案**:
- Spring AI: API抽象不够灵活，学习成本高，版本迭代快不稳定
- ChatGPT: 需要翻墙，国内不可用，监管风险
- 文心一言: 效果略逊于Qwen，API一致性不如Qwen

### 2.3 缓存选型

| 选项 | 选择 | 原因 |
|------|------|------|
| Redis + Caffeine (二级缓存) | **Caffeine(本地) + Redis(分布式)** | 二级缓存策略，本地缓存减少网络开销，Redis支持分布式部署和锁 |
| Redisson vs Jedis | **Jedis + 自封装** | Jedis更轻量，Redisson功能太重、学习曲线陡峭 |

**为什么不选其他方案**:
- 单用Redis: 每次缓存访问都要网络开销，高并发下Redis成为瓶颈
- Ehcache: 不支持分布式，无法在集群环境共享
- 不用缓存: 数据库压力过大，接口响应慢

### 2.4 向量数据库选型

| 选项 | 选择 | 原因 |
|------|------|------|
| Elasticsearch vs Milvus vs Pinecone vs Qdrant | **Elasticsearch** | 自托管成熟、功能完善、支持混合检索(关键词+向量)、IK中文分词、无需额外部署 |
| DashVector vs 自托管ES | **自托管ES** | 免费可控、支持复杂查询、集成现有MySQL数据、运维统一 |

**为什么不选其他方案**:
- Milvus: 需要单独部署，运维成本高，资源占用大
- Pinecone: 云服务有成本和数据出境顾虑，国内项目不适合
- Qdrant: Rust开发，排查问题困难，社区相对较小
- Chroma: 单机版，生产环境不推荐

### 2.5 定时任务选型

| 选项 | 选择 | 原因 |
|------|------|------|
| Quartz vs @Scheduled vs XXL-Job | **Quartz** | 功能完善、分布式支持、Spring原生集成、Job持久化 |

**为什么不选其他方案**:
- Spring @Scheduled: 无分布式支持，集群环境会重复执行
- XXL-Job: 需要额外部署服务，小项目不值得
- 消息队列延迟: 复杂度过高，不适合定时场景

// 向量检索核心逻辑
public List<Map<String, Object>> vectorSearch(String query, String school, int topK) {
    // 1. 生成查询向量
    float[] queryVector = embedText(query);

    // 2. 构建KNN查询 + 学校过滤
    SearchRequest request = SearchRequest.of(s -> s
        .index(INDEX_NAME)
        .size(topK)
        .query(q -> q.bool(b -> b
            .filter(f -> f.term(t -> t.field("schoolWebsite").value(school)))
            .should(sq -> sq.knn(k -> k
                .field("embedding")
                .queryVector(queryVector)
                .k(topK)
            ))
        ))
    );

    // 3. 执行搜索
    SearchResponse<Map> response = esClient.search(request, Map.class);

    // 4. 格式化结果
    return formatEsResults(response.hits().hits());
}

public CommonResult<?> importSchedule(Long userId, MultipartFile file) {
    String lowerName = file.getOriginalFilename().toLowerCase();

    // 分支1: Excel直接解析
    if (lowerName.endsWith(".xls") || lowerName.endsWith(".xlsx")) {
        return importFromExcel(userId, file);
    }

    // 分支2: AI智能解析 (PDF/Word/TXT)
    if (lowerName.endsWith(".pdf") || lowerName.endsWith(".doc") ||
        lowerName.endsWith(".docx") || lowerName.endsWith(".txt")) {
        return importFromDocument(userId, file);
    }

    return CommonResult.fail("不支持的文件格式");
}

// AI智能导入流程
private CommonResult<?> importFromDocument(Long userId, MultipartFile file) {
    // 1. 解析文档为文本块
    List<String> chunks = documentParserService.parseDocument(file);
    String rawText = String.join("\n", chunks);

    // 2. 构建AI解析Prompt
    String prompt = String.format("""
        你是一个课程表解析助手。
        请将以下课表文本解析为JSON数组:
        [{\"name\":\"课程名\",\"weekday\":1,\"startTime\":\"08:00\",\"endTime\":\"09:40\",\"teacher\":\"教师名\"},...]

        课表文本:
        %s
        """, rawText);

    // 3. 调用AI解析
    String jsonResponse = qingTuAgent.chat(prompt);

    // 4. 提取JSON数组
    jsonResponse = extractJsonArray(jsonResponse);

    // 5. 反序列化并保存
    CourseImportDTO[] coursesArray = objectMapper.readValue(jsonResponse, CourseImportDTO[].class);
    courseCacheService.saveCourses(userId, Arrays.asList(coursesArray));

    return CommonResult.success(Map.of("message", "导入成功", "count", coursesArray.length));
}

public class RedisLockUtil {
    public boolean tryLock(String lockKey, int expireSeconds) {
        return Boolean.TRUE.equals(
            redisTemplate.opsForValue()
                .setIfAbsent(lockKey, "1", expireSeconds, TimeUnit.SECONDS)
        );
    }

    public void unlock(String lockKey) {
        redisTemplate.delete(lockKey);
    }
}

// Job中使用
@Override
public void execute(JobExecutionContext context) {

        } finally {
            RedisLockUtil.unlock(lockKey);
        }
    }
}

public CommonResult<?> executeTool(Long userId, String toolName, Map<String, Object> params) {
    // 1. 获取工具定义
    ToolHandlerData tool = toolRegistry.getTool(toolName);
    if (tool == null) {
        return CommonResult.fail("工具不存在: " + toolName);
    }

        return CommonResult.fail("权限不足");

    long startTime = System.currentTimeMillis();
    Object result;

// 工具注册
@Component
public class ToolRegistry {

        toolHandlers.put(name, new ToolHandlerData(name, handler, permission));
    }

// 新增工具只需注册
@PostConstruct
public void init() {
    toolRegistry.register("school_search",
        params -> schoolTools.search(params.get("keyword")),
